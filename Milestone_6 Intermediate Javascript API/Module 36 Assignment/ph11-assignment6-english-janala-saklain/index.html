1. What is the difference between var, let, and const?
The differences between var, let, and const:
  • var is more flexible compared to let and const. It has function scope or global scope, meaning that a variable declared with var is accessible within the entire function in which it is defined. However, var does not have block scope, so it can lead to unexpected behavior when used inside loops or conditional statements.
  • let was introduced in ES6 as a replacement for var. It has block scope, which means that a variable declared with let is only accessible within the block (a pair of curly braces) in which it is defined. This can prevent unexpected variable hoisting and makes the code more predictable.
  • const also has block scope like let, but it is a constant variable that cannot be reassigned after it is defined. It is useful when you have a value that should not change throughout the program. However, it is important to note that const does not make objects or arrays immutable. While you cannot reassign the entire object or array, you can modify its properties or elements.`

    • 3. What are arrow functions, and how are they different from regular functions?
    অ্যারো ফাংশন (Arrow functions) এবং সাধারণ ফাংশনের (Regular functions) মধ্যে প্রধান পার্থক্যগুলো হলো সিনট্যাক্স, this এর ব্যবহার, arguments অবজেক্টের অনুপস্থিতি, এবং কনস্ট্রাক্টর হিসেবে ব্যবহার না করা। অ্যারো ফাংশন ES6-এ প্রবর্তিত, যা কোডকে সংক্ষিপ্ত ও পাঠযোগ্য করে তোলে।
    সিনট্যাক্সের দিক থেকে, অ্যারো ফাংশন সাধারণ ফাংশনের তুলনায় অনেক সংক্ষিপ্ত। 'function' কীওয়ার্ডের পরিবর্তে '=>' চিহ্ন ব্যবহার করা হয়। this এর ব্যবহারে একটি গুরুত্বপূর্ণ পার্থক্য রয়েছে। সাধারণ ফাংশনে this এর মান ফাংশন কল করার পদ্ধতির উপর নির্ভর করে, যা বিভ্রান্তিকর হতে পারে। অ্যারো ফাংশনে this ল্যাক্সিক্যাল স্কোপ থেকে আসে, অর্থাৎ যেখানে ফাংশনটি লেখা হয়েছে সেখানকার this ব্যবহৃত হয়, যা কলব্যাক ও ইভেন্ট হ্যান্ডলারের জন্য সুবিধা প্রদান করে।
    অ্যারো ফাংশনে arguments অবজেক্ট থাকে না, এর পরিবর্তে রেস্ট প্যারামিটার ব্যবহার করা যায়। অ্যারো ফাংশন কনস্ট্রাক্টর হিসেবে ব্যবহার করা যায় না, অর্থাৎ new কীওয়ার্ড ব্যবহার করে অবজেক্ট তৈরি করা যায় না। এক লাইনের অ্যারো ফাংশনে ইম্প্লিসিট রিটার্ন সুবিধা রয়েছে, যা কোডকে আরও সংক্ষিপ্ত করে। এছাড়াও অ্যারো ফাংশনে প্রোটোটাইপ প্রোপার্টি নেই।
    4. How do JavaScript Promises work?
    জাভাস্ক্রিপ্ট প্রমিজ (Promises) হলো ভবিষ্যতে কোনো কাজ শেষ হলে কী হবে তা জানানোর একটি পদ্ধতি। এটা অনেকটা প্রতিশ্রুতির মতো, যেখানে বলা হয় কাজটা হলে কী ফল পাওয়া যাবে বা না হলে কী হবে। প্রমিজ তিন রকম হতে পারে: পেন্ডিং (pending), ফুলফিল্ড (fulfilled), আর রিজেক্টেড (rejected)।
    যখন কোনো কাজ শুরু হয়, তখন প্রমিজ পেন্ডিং থাকে। কাজটা শেষ হলে, যদি ঠিকঠাক হয়, তাহলে ফুলফিল্ড হয়, আর কোনো সমস্যা হলে রিজেক্টেড হয়। ফুলফিল্ড হলে .then() ব্যবহার করে ফলাফল পাওয়া যায়, আর রিজেক্টেড হলে .catch() ব্যবহার করে ত্রুটি ধরা যায়। প্রমিজ অ্যাসিঙ্ক্রোনাস (asynchronous) কাজ সহজ করে, মানে কোড না থেমে অন্যান্য কাজ করতে পারে।
    5. How does closure work in JavaScript
    জাভাস্ক্রিপ্টে ক্লোজার (Closure) হলো এমন একটি বৈশিষ্ট্য যেখানে একটি ফাংশন তার বাইরের (প্যারেন্ট) ফাংশনের ভেরিয়েবলগুলোকে মনে রাখে এবং ব্যবহার করতে পারে, এমনকি প্যারেন্ট ফাংশনটি তার কাজ শেষ করে দিলেও।
    সহজভাবে বললে, যখন একটি ফাংশনের ভেতর আরেকটি ফাংশন তৈরি করা হয়, তখন ভেতরের ফাংশনটি বাইরের ফাংশনের ভেরিয়েবলগুলোর একটি "ক্লোজড ওভার" পরিবেশ তৈরি করে। প্যারেন্ট ফাংশনটি শেষ হয়ে গেলেও, ভেতরের ফাংশনটি সেই ভেরিয়েবলগুলোকে অ্যাক্সেস করতে পারে।
    এটা অনেকটা এমন, যেন একটি বাক্স তৈরি করা হলো, যার ভেতরে কিছু জিনিস রাখা হলো। বাইরের বাক্সটি সরিয়ে নিলেও, ভেতরের বাক্সটি তার ভেতরের জিনিসগুলো ধরে রাখে। ক্লোজার জাভাস্ক্রিপ্টে অ্যাসিঙ্ক্রোনাস কোড, ইভেন্ট হ্যান্ডলার এবং প্রাইভেট ভেরিয়েবল তৈরিতে খুব কাজে লাগে।
